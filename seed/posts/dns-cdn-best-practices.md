# 域名解析与 CDN 加速最佳实践

域名解析和 CDN 加速是现代网站架构的基础设施组合。本文将介绍如何将两者结合，最大化网站性能和可用性。

## 一、域名解析策略

### 1.1 解析线路优化

利用 DNS 服务商的**智能解析**功能，根据用户地理位置返回不同的 IP：

- **电信用户** → 电信机房 IP
- **联通用户** → 联通机房 IP
- **海外用户** → 海外节点 IP

### 1.2 负载均衡

通过 DNS 轮询实现简单负载均衡：

```
example.com  A  1.1.1.1  (权重 10)
example.com  A  2.2.2.2  (权重 10)
example.com  A  3.3.3.3  (权重 10)
```

> 注意：DNS 轮询无法感知服务器健康状态，建议配合专业的负载均衡器使用。

### 1.3 TTL 配置建议

| 场景 | 推荐 TTL | 说明 |
|------|---------|------|
| 稳定生产环境 | 600-3600s | 减少 DNS 查询次数，降低延迟 |
| 即将迁移 | 60-120s | 提前降低 TTL，切换时快速生效 |
| 故障切换 | 30-60s | 紧急情况下的最小 TTL |

## 二、CDN 加速原理

CDN（内容分发网络）通过在全球部署边缘节点，将内容缓存到离用户最近的位置：

1. 用户请求资源（如一张图片）
2. DNS 解析将用户导向最近的 CDN 节点
3. 如果节点有缓存，直接返回（缓存命中）
4. 如果没有缓存，节点回源站拉取并缓存（缓存未命中）

## 三、CDN 配置步骤

### 3.1 添加加速域名

以 Cloudflare 为例：

1. 添加站点，选择 Free 计划
2. 将域名 NS 记录改为 Cloudflare 提供的 DNS 服务器
3. 等待 DNS 生效（24-48 小时）

### 3.2 缓存规则配置

```
# 静态资源：长时间缓存
*.js       → TTL 7 天
*.css      → TTL 7 天
*.png      → TTL 30 天
*.ico      → TTL 30 天

# 动态内容：不缓存或短时间缓存
/api/*     → TTL 0
/index.htm → TTL 600s
```

### 3.3 HTTPS 配置

CDN 层建议强制 HTTPS：

1. 在 CDN 控制台开启 SSL/TLS
2. 选择 Full（严格）模式 — CDN 到源站也使用 HTTPS
3. 开启 HTTP 自动跳转到 HTTPS

## 四、域名 + CDN + SSL 协同架构

一个完整的生产环境架构：

```
用户 ──HTTPS──▶ CDN 边缘节点 ──HTTPS──▶ 源站 Nginx ──▶ 应用服务
                    │
                    ├── 缓存静态资源
                    ├── DDoS 防护
                    ├── WAF 规则
                    └── HTTP/2 + Brotli 压缩
```

### 配置要点

1. **CDN 证书** — 使用 CDN 提供的证书管理，自动续签
2. **源站证书** — 使用 Let's Encrypt 或云厂商免费证书
3. **回源方式** — 建议全链路 HTTPS（CDN 配置 Full Strict）
4. **缓存策略** — 静态资源 CDN 缓存，动态请求透传

## 五、性能优化实践

1. **开启 Brotli 压缩** — 比 Gzip 压缩率高 20-30%
2. **配置 HTTP/2** — 多路复用减少连接数
3. **使用 OCSP Stapling** — 提升 TLS 握手速度
4. **开启 HSTS Preload** — 浏览器内置 HTTPS 强制列表

## 六、常见问题

**Q：配置 CDN 后网站无法访问？**
A：检查 DNS 解析是否指向 CDN 提供的 CNAME，且源站允许 CDN 回源 IP 访问。

**Q：CDN 缓存不生效怎么办？**
A：配置 CDN 的回源 HOST 参数，确保 CDN 请求头中的 Host 与实际访问域名一致。

**Q：HTTPS 配置后出现混合内容警告？**
A：页面中所有资源（图片、JS、CSS）必须使用 HTTPS 链接，检查是否有 `http://` 的引用。
